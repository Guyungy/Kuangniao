# 🗄 完整数据库表设计（产品文档）

## 1. 会员表（`members`）

存储所有会员信息，作为充值、订单的核心关联。

| 字段名             | 类型            | 必填 | 说明                  | 备注                |
| --------------- | ------------- | -- | ------------------- | ----------------- |
| id              | INT PK        | ✅  | 主键，自增 ID            | 系统唯一标识            |
| nickname        | VARCHAR(50)   | ✅  | 会员昵称                | 展示用               |
| phone           | VARCHAR(20)   | ❌  | 手机号                 | 搜索/联系方式           |
| balance         | DECIMAL(10,2) | ✅  | 当前余额                | 系统计算（累计充值 - 累计消费） |
| total\_recharge | DECIMAL(10,2) | ✅  | 累计充值金额              | 系统自动累加            |
| total\_consume  | DECIMAL(10,2) | ✅  | 累计消费金额              | 系统自动累加            |
| status          | ENUM          | ✅  | 状态（active/disabled） | 可冻结会员             |
| created\_at     | TIMESTAMP     | ✅  | 注册时间                | 系统自动生成            |
| updated\_at     | TIMESTAMP     | ✅  | 最近更新时间              | 系统自动更新            |

---

## 2. 充值表（`recharges`）

记录所有充值流水。

| 字段名         | 类型            | 必填 | 说明                 | 备注                |
| ----------- | ------------- | -- | ------------------ | ----------------- |
| id          | INT PK        | ✅  | 主键，自增 ID           | 系统唯一标识            |
| member\_id  | INT FK        | ✅  | 会员 ID              | 外键关联 `members.id` |
| amount      | DECIMAL(10,2) | ✅  | 充值金额               |                   |
| method      | ENUM          | ✅  | 支付方式（balance/scan） | 扫码充值 / 余额充值       |
| created\_at | TIMESTAMP     | ✅  | 充值时间               | 系统自动生成            |
| remark      | VARCHAR(255)  | ❌  | 备注（可选）             | 管理员手动填写           |

---

## 3. 订单表（`orders`）

记录所有消费订单。

| 字段名           | 类型            | 必填 | 说明                       | 备注                |
| ------------- | ------------- | -- | ------------------------ | ----------------- |
| id            | INT PK        | ✅  | 主键，自增 ID                 | 系统唯一标识            |
| member\_id    | INT FK        | ✅  | 会员 ID                    | 外键关联 `members.id` |
| worker\_id    | INT FK        | ✅  | 打手 ID                    | 外键关联 `workers.id` |
| duration      | DECIMAL(5,2)  | ✅  | 服务时长（小时）                 | 可录入 1.5 小时        |
| price\_origin | DECIMAL(10,2) | ✅  | 原价（时长 × 单价）              | 系统自动计算            |
| discount      | DECIMAL(10,2) | ❌  | 优惠金额                     | 管理员可手动填写          |
| price\_final  | DECIMAL(10,2) | ✅  | 实付金额                     | = 原价 - 优惠         |
| pay\_method   | ENUM          | ✅  | 支付方式（balance/scan/mixed） | 余额 / 扫码 / 混合支付    |
| pay\_balance  | DECIMAL(10,2) | ❌  | 使用余额金额                   | 混合支付时用            |
| pay\_scan     | DECIMAL(10,2) | ❌  | 扫码支付金额                   | 混合支付时用            |
| remark        | VARCHAR(255)  | ❌  | 管理员备注（优惠原因/特殊情况说明）       |                   |
| created\_at   | TIMESTAMP     | ✅  | 下单时间                     | 系统自动生成            |
| updated\_at   | TIMESTAMP     | ✅  | 最近更新时间                   | 系统自动更新            |

---

## 4. 打手表（`workers`）

维护所有打手信息，含实名与结算资料。

| 字段名           | 类型            | 必填 | 说明              | 备注               |
| ------------- | ------------- | -- | --------------- | ---------------- |
| id            | INT PK        | ✅  | 主键，自增 ID        | 系统唯一标识           |
| name          | VARCHAR(50)   | ✅  | 打手昵称            | 展示用              |
| real\_name    | VARCHAR(50)   | ✅  | 真实姓名            | 财务结算用            |
| id\_number    | VARCHAR(18)   | ✅  | 身份证号            | 风控 & 结算          |
| phone         | VARCHAR(20)   | ✅  | 联系电话            | 内部联系             |
| wechat\_id    | VARCHAR(50)   | ❌  | 微信号             | 辅助联系             |
| bank\_account | VARCHAR(50)   | ❌  | 银行卡号            | 财务结算用            |
| bank\_name    | VARCHAR(100)  | ❌  | 开户行名称           | 例如：中国工商银行深圳南山支行  |
| price\_hour   | DECIMAL(10,2) | ✅  | 每小时单价           | 默认用于订单价格计算       |
| type          | ENUM          | ✅  | 类型（跑刀/陪玩/陪练/其他） | 用于分类             |
| status        | ENUM          | ✅  | 状态（待审核/在职/禁用）   | 审核通过后才能出现在下单页面   |
| created\_at   | TIMESTAMP     | ✅  | 添加时间            | 系统自动生成           |
| updated\_at   | TIMESTAMP     | ✅  | 最近更新时间          | 系统自动更新           |
| remark        | VARCHAR(255)  | ❌  | 管理员备注           | 备注信息（例如：合作协议编号等） |

## 5. 推送日志表（`wecom_logs`）

记录每次企业微信群播报的内容，便于追溯。

| 字段名         | 类型        | 必填 | 说明                           | 备注     |
| ----------- | --------- | -- | ---------------------------- | ------ |
| id          | INT PK    | ✅  | 主键，自增 ID                     | 系统唯一标识 |
| type        | ENUM      | ✅  | 消息类型（recharge/order/warning） |        |
| content     | TEXT      | ✅  | 推送内容（JSON 或 Markdown）        | 保存推送文本 |
| created\_at | TIMESTAMP | ✅  | 推送时间                         | 系统自动生成 |

---

# 🔄 核心业务流转

## 充值流程

1. 管理员录入充值（选择会员 + 金额 + 支付方式）
2. 插入 `recharges`
3. 更新 `members.balance` & `total_recharge`
4. 生成企业微信推送 → 记录 `wecom_logs`

---

## 下单流程

1. 管理员录入订单（会员 + 打手 + 时长）
2. 系统计算原价 → 管理员可改价（优惠）
3. 选择支付方式（余额 / 扫码 / 混合支付）
4. 扣费 & 插入 `orders`
5. 更新 `members.balance` & `total_consume`
6. 生成企业微信推送 → 记录 `wecom_logs`

---

## 自动提醒

* **余额不足**：

  * 下单时检测余额 < price\_final → 前端提示
  * 系统同时生成群播报（余额不足提醒）

* **打手接单超时/超额**：

  * 定时任务扫描 `orders`，检测打手单日接单时长 > 8h → 群播报提醒
