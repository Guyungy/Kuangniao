# 🎯 一、系统结构（模块划分）

后台只有一个角色：**管理员**
核心模块 5 个：

1. **会员管理**

   * 查看会员信息（昵称 / 手机号 / 余额 / 累计充值 / 累计消费）
   * 新增 / 编辑 / 删除会员
   * 查看流水（充值记录、消费记录）

2. **充值管理**

   * 新增充值（选择会员 + 金额）
   * 系统自动更新余额
   * 群里自动播报「充值成功」

3. **订单管理（消费）**

   * 新增订单

     * 选择会员、打手
     * 填写服务时长 → 系统自动算价格
     * 付款方式选择：余额 / 扫码 / 混合
     * 可修改价格（优惠）
     * 确认下单 → 系统扣款 / 记账
   * 群里自动播报「订单完成」

4. **打手管理**（仅维护信息）

   * 昵称 / 类型 / 单价 / 状态
   * 下单时从这里选择

5. **消息推送（企业微信）**

   * 充值消息
   * 消费消息（含付款方式 / 优惠信息）
   * 余额不足提醒
   * 打手接单统计提醒（可选）

# 🔄 二、流程设计

## 1. 充值录入流程

```
管理员 → 充值管理 → 新增充值 → 选择会员 → 输入金额 → 确认
      ↓
系统更新余额 & 充值流水
      ↓
企业微信群播报：
💰充值成功
会员：张三
金额：¥3000
当前余额：¥29750
时间：2025-08-22 15:21
```

---

## 2. 下单消费流程

```
管理员 → 订单管理 → 新增订单
      ↓
选择会员 + 打手 → 输入服务时长（自动算价格）
      ↓
选择付款方式：
  - 余额支付（余额足 → 扣款）
  - 扫码支付（不动余额，只记账）
  - 混合支付（余额不足 → 扫码补差额）
      ↓
管理员可修改金额（优惠），需填写原因
      ↓
确认 → 生成订单流水 & 更新余额/消费
      ↓
企业微信群播报：
✅订单完成
会员：Liszt
打手：李美美
时长：2h
原价：¥300
优惠：¥50
实付：¥250
付款方式：余额 ¥100 + 扫码 ¥150
当前余额：¥0
时间：2025-08-22 18:10
```

---

## 3. 余额不足提醒流程

```
管理员 → 尝试下单 → 系统检测余额
      ↓
余额不足 → 提示“余额不足，请充值”
      ↓
群播报（可选）：
⚠️余额不足提醒
会员：张三
当前余额：¥80
请及时充值
```

---

## 4. 自动提醒流程（智能规则）

* **充值提醒**：每笔充值自动播报
* **消费提醒**：每笔消费自动播报
* **优惠提醒**：订单价格被修改时，群消息自动带「优惠」字段
* **余额预警**：余额低于阈值（如 100 元），自动推送提醒
* **打手提醒（可选）**：某打手今日接单超 N 小时，群播报

# 📲 三、页面交互（后台）

### 首页

* 快捷入口：会员 / 充值 / 下单 / 报表
* 今日概览：充值额 / 消费额 / 余额不足人数

### 会员管理

* 列表：昵称、手机号、余额、累计充值、累计消费
* 搜索 / 详情（包含流水记录）

### 充值管理

* 输入会员 + 金额 → 确认
* 系统播报消息

### 订单管理

* 输入：会员 + 打手 + 时长
* 自动计算金额 → 选择付款方式 → 可修改金额（优惠）
* 确认下单 → 系统播报

# 🧠 四、整体逻辑总结

1. **核心闭环**：充值 → 余额更新 → 消费 → 扣费 → 消息推送
2. **灵活性**：支持余额 / 扫码 / 混合支付，支持人工改价
3. **透明性**：所有操作群里播报（原价 / 优惠 / 实付 / 付款方式）
4. **智能化**：余额不足、优惠、打手工时 → 自动提醒
5. **轻量化**：一个后台入口，一个管理员账号，企业微信群作为前端展示
