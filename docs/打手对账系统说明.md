# 打手对账系统使用说明

## 系统概述

打手对账系统是一个专门用于管理打手工资结算和核账的财务管理系统，类似于工资条功能。系统可以记录每个打手在特定时间段内的订单情况、应得金额、实发金额等信息，并提供核账确认和争议处理功能。

## 主要功能

### 1. 对账记录管理
- **新增对账记录**：为指定打手创建特定时间段的结算记录
- **编辑对账记录**：修改未核账的记录信息
- **删除对账记录**：删除未核账的记录
- **查看对账详情**：查看完整的对账信息和相关订单

### 2. 结算类型
- **日结**：按天进行结算
- **周结**：按周进行结算
- **月结**：按月进行结算

### 3. 核账状态管理
- **待核账**：新创建的对账记录，等待财务人员确认
- **已核账**：财务人员已确认的对账记录
- **有争议**：存在争议的对账记录，需要进一步处理

### 4. 操作记录
- 记录核账操作人
- 记录核账时间
- 记录核账备注
- 记录争议原因

## 系统架构

### 后端组件
- **WorkerSettlement模型**：对账记录的数据模型
- **对账API路由**：提供完整的CRUD操作和核账功能
- **数据库表**：`worker_settlements`表存储所有对账数据

### 前端组件
- **对账管理页面**：主要的管理界面
- **对账API接口**：与后端通信的接口层
- **表单组件**：新增、编辑对账记录的表单

## 使用流程

### 1. 创建对账记录
1. 进入"打手对账"页面
2. 点击"新增对账"按钮
3. 选择打手、结算类型、时间范围
4. 输入实发金额
5. 保存对账记录

### 2. 核账确认
1. 在待核账记录中点击"确认核账"
2. 输入核账备注（可选）
3. 确认核账操作

### 3. 争议处理
1. 在待核账记录中点击"标记争议"
2. 输入争议原因
3. 确认标记争议

### 4. 查询和筛选
- 按打手筛选
- 按状态筛选
- 按结算类型筛选
- 按时间范围筛选

## 数据字段说明

### 基础信息
- **打手信息**：姓名、真实姓名、联系电话
- **结算信息**：结算类型、开始日期、结束日期
- **订单统计**：订单数量、订单总金额、总小时数

### 金额信息
- **小时单价**：打手的小时工资标准
- **应得金额**：根据工作小时数计算的应得工资
- **实发金额**：实际发放的工资金额
- **差额**：实发金额与应得金额的差值
- **差额说明**：对差额的解释说明

### 状态信息
- **核账状态**：待核账、已核账、有争议
- **操作记录**：操作人、操作时间、备注信息

## 数据库表结构

```sql
CREATE TABLE worker_settlements (
  id INT AUTO_INCREMENT PRIMARY KEY COMMENT '对账ID',
  worker_id INT NOT NULL COMMENT '打手ID',
  settlement_type ENUM('daily', 'weekly', 'monthly') NOT NULL COMMENT '结算类型',
  start_date DATE NOT NULL COMMENT '结算开始日期',
  end_date DATE NOT NULL COMMENT '结算结束日期',
  order_count INT NOT NULL DEFAULT 0 COMMENT '订单数量',
  order_amount DECIMAL(10,2) NOT NULL DEFAULT 0.00 COMMENT '订单总金额',
  total_hours DECIMAL(10,2) NOT NULL DEFAULT 0.00 COMMENT '小时数',
  hourly_rate DECIMAL(10,2) NOT NULL DEFAULT 0.00 COMMENT '小时单价',
  expected_amount DECIMAL(10,2) NOT NULL DEFAULT 0.00 COMMENT '应得金额',
  actual_amount DECIMAL(10,2) NOT NULL DEFAULT 0.00 COMMENT '实发金额',
  difference_amount DECIMAL(10,2) NOT NULL DEFAULT 0.00 COMMENT '差额',
  difference_reason TEXT NULL COMMENT '差额说明',
  status ENUM('pending', 'confirmed', 'disputed') NOT NULL DEFAULT 'pending' COMMENT '核账状态',
  confirmed_by INT NULL COMMENT '核账操作人ID',
  confirmed_at DATETIME NULL COMMENT '核账时间',
  confirmation_note TEXT NULL COMMENT '核账备注',
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
);
```

## API接口

### 主要接口
- `GET /api/v1/worker-settlements/page` - 获取对账分页列表
- `GET /api/v1/worker-settlements/:id` - 获取对账详情
- `POST /api/v1/worker-settlements` - 创建对账记录
- `PUT /api/v1/worker-settlements/:id` - 更新对账记录
- `PUT /api/v1/worker-settlements/:id/confirm` - 确认核账
- `PUT /api/v1/worker-settlements/:id/dispute` - 标记争议
- `DELETE /api/v1/worker-settlements/:id` - 删除对账记录

- 重启后端服务

- 重新构建前端项目

## 注意事项

1. **权限控制**：只有具有财务权限的用户才能进行核账操作
2. **数据完整性**：已核账的记录不能修改或删除
3. **时间范围**：确保结算时间范围不与现有记录重叠
4. **金额计算**：系统会自动计算应得金额和差额
5. **操作记录**：所有核账操作都会记录操作人和时间
