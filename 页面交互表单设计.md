# 🖥️ 页面交互表单设计

## 1. 会员管理表单

### 新增 / 编辑会员

| 字段   | 必填 | 输入方式    | 说明                | 交互逻辑       |
| ---- | -- | ------- | ----------------- | ---------- |
| 昵称   | ✅  | 文本输入框   | 会员昵称              | 可重复，不做唯一约束 |
| 手机号  | ❌  | 文本输入框   | 会员联系方式            | 可用于搜索      |
| 状态   | ✅  | 下拉选择    | active / disabled | 默认 active  |
| 余额   | ❌  | 只读（自动算） | 当前余额              | 不能手动改      |
| 累计充值 | ❌  | 只读      | 系统自动计算            | —          |
| 累计消费 | ❌  | 只读      | 系统自动计算            | —          |

👉 列表页面：支持按「昵称 / 手机号」搜索，点击会员 → 查看详情页（Tabs：充值记录 / 消费记录）。

## 2. 充值管理表单

### 新增充值

| 字段   | 必填 | 输入方式  | 说明             | 交互逻辑        |
| ---- | -- | ----- | -------------- | ----------- |
| 会员   | ✅  | 下拉搜索框 | 选择会员           | 昵称 + 手机号搜索  |
| 金额   | ✅  | 数字输入框 | 充值金额           | 必须 >0       |
| 支付方式 | ✅  | 下拉选择  | balance / scan | 余额充值 / 扫码充值 |
| 备注   | ❌  | 文本输入框 | 可写入管理员备注       | —           |

👉 提交成功后：

* 更新余额 → 余额 += 金额
* 写入充值流水
* **企业微信群推送：充值成功**

## 3. 订单管理表单

### 新增订单

| 字段   | 必填 | 输入方式    | 说明                     | 交互逻辑              |
| ---- | -- | ------- | ---------------------- | ----------------- |
| 会员   | ✅  | 下拉搜索框   | 选择会员                   | 昵称 + 手机号搜索        |
| 打手   | ✅  | 下拉搜索框   | 选择打手                   | 仅显示在职打手           |
| 服务时长 | ✅  | 数字输入框   | 小时数（可输入 1.5）           | 系统自动算原价           |
| 原价   | ✅  | 只读（自动算） | 时长 × 打手单价              | 系统计算              |
| 实付金额 | ✅  | 可修改数字框  | 默认 = 原价，可手动改（优惠/特殊情况）  | 改动后记录优惠额          |
| 优惠原因 | ❌  | 文本输入框   | 如果改价必须填写原因             | —                 |
| 支付方式 | ✅  | 下拉选择    | balance / scan / mixed | 余额 / 扫码 / 混合      |
| 余额支付 | ❌  | 数字输入框   | 仅混合支付时显示               | 系统校验 balance 是否足够 |
| 扫码支付 | ❌  | 数字输入框   | 仅混合支付时显示               | 自动算出差额，可修改        |
| 备注   | ❌  | 文本输入框   | 订单说明                   | —                 |

👉 提交成功后：

* **余额支付** → 扣余额
* **扫码支付** → 只记账，不动余额
* **混合支付** → 扣余额 + 扫码补差额
* 更新消费金额 → `total_consume`
* 写入订单流水
* **企业微信群推送：订单完成**

## 4. 打手管理表单

### 新增 / 编辑打手

| 字段    | 必填 | 输入方式  | 说明              | 交互逻辑            |
| ----- | -- | ----- | --------------- | --------------- |
| 昵称    | ✅  | 文本输入框 | 打手昵称            | 展示用             |
| 真实姓名  | ✅  | 文本输入框 | 打手实名            | 结算用             |
| 身份证号  | ✅  | 文本输入框 | 打手身份证号          | 脱敏显示（前端只显示前6后4） |
| 手机号   | ✅  | 文本输入框 | 联系电话            | 内部联系            |
| 微信号   | ❌  | 文本输入框 | 备用联系方式          | —               |
| 银行卡号  | ❌  | 文本输入框 | 结算用             | 脱敏显示（只显示末尾4位）   |
| 开户行名称 | ❌  | 文本输入框 | 例如：中国工商银行深圳南山支行 | —               |
| 单价/H  | ✅  | 数字输入框 | 每小时单价           | 下单时计算原价用        |
| 类型    | ✅  | 下拉选择  | 跑刀 / 陪玩 / 其他    | —               |
| 状态    | ✅  | 下拉选择  | 待审核 / 在职 / 禁用   | 默认待审核，审核后可下单    |
| 备注    | ❌  | 文本输入框 | 特殊说明（合作编号等）     | —               |

## 5. 报表中心（无表单，展示页面）

* **今日概览**：今日充值总额 / 今日消费总额 / 活跃会员数
* **趋势图**：充值趋势、消费趋势（按日/周/月）
* **排行榜**：

  * 会员余额 Top5
  * 打手接单量 Top5

👉 全部只读展示，由后端 API 提供数据。

# 📲 自动提醒逻辑嵌入表单交互

* **订单提交**：

  * 如果余额不足 → 前端弹窗提示「余额不足，请先充值」
  * 如果管理员改价 → 强制填写优惠原因
* **充值提交**：

  * 如果充值前余额 < 阈值（如 ¥100）→ 群播报时自动标注「低余额补充」
* **打手录入**：

  * 身份证、银行卡保存时 → 前端展示，后端原文存储
