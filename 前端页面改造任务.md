# 🖥 前端页面改造任务（基于 叨叨电竞）

### 🔹 全局改造

* 删除：角色管理、部门管理、字典管理、系统配置等复杂模块
* 保留：登录页、首页（Dashboard）、表格、表单、图表组件
* 菜单结构调整为：

  * 会员管理
  * 充值管理
  * 订单管理
  * 打手管理
  * 报表中心

---

### 🔹 会员管理

* 列表页：

  * 字段：昵称 / 手机号 / 余额 / 累计充值 / 累计消费 / 注册时间
  * 操作：新增、编辑、删除、查看详情
* 详情页：

  * Tab1：充值记录
  * Tab2：消费记录

---

### 🔹 充值管理

* 列表页：

  * 字段：充值编号 / 会员 / 金额 / 支付方式 / 时间
* 表单页（新增充值）：

  * 选择会员（下拉搜索）
  * 输入金额
  * 选择支付方式（余额充值 / 扫码充值）
  * 提交 → 调用 `/api/recharge`

---

### 🔹 订单管理

* 列表页：

  * 字段：订单编号 / 会员 / 打手 / 服务时长 / 原价 / 实付 / 优惠 / 支付方式 / 时间
* 表单页（新增订单）：

  * 选择会员
  * 选择打手（下拉搜索）
  * 输入服务时长 → 系统自动计算价格（时长 × 单价）
  * 可修改金额（优惠）+ 输入优惠原因
  * 选择支付方式（余额 / 扫码 / 混合支付）
  * 提交 → 调用 `/api/order`

---

### 🔹 打手管理

* 列表页：

  * 字段：打手昵称 / 联系方式 / 类型 / 单价 / 状态
* 表单页：新增 / 编辑打手信息

---

### 🔹 报表中心

* 使用 ECharts 展示：

  * 今日充值额 / 今日消费额
  * 充值趋势图（按日/周/月）
  * 消费趋势图（按日/周/月）
  * 会员余额排行 Top5
  * 打手接单统计排行

# ⚙️ 后端 API 任务（基于 Node.js + NestJS）

### 🔹 会员 API

* `GET /api/members` 获取会员列表
* `GET /api/members/:id` 获取会员详情（含充值、消费流水）
* `POST /api/members` 新增会员
* `PUT /api/members/:id` 修改会员信息
* `DELETE /api/members/:id` 删除会员

---

### 🔹 充值 API

* `POST /api/recharge` 新增充值

  * 入参：会员ID、金额、支付方式
  * 逻辑：更新会员余额 & 累计充值
  * 推送：企业微信群「充值成功」消息
* `GET /api/recharge` 查询充值记录

---

### 🔹 订单 API

* `POST /api/order` 新增订单

  * 入参：会员ID、打手ID、时长、原价、实付金额、支付方式、优惠金额+原因
  * 逻辑：

    * 如果余额支付 → 扣减余额
    * 如果扫码支付 → 只记账，不动余额
    * 如果混合支付 → 扣余额 + 扫码补差额
    * 记录原价 / 优惠 / 实付金额
  * 推送：企业微信群「订单完成」消息
* `GET /api/order` 查询订单记录

---

### 🔹 打手 API

* `GET /api/workers` 获取打手列表
* `POST /api/workers` 新增打手
* `PUT /api/workers/:id` 修改打手信息
* `DELETE /api/workers/:id` 删除打手

---

### 🔹 报表 API

* `GET /api/report/overview` 获取今日数据（充值总额 / 消费总额 / 活跃会员数）
* `GET /api/report/recharge` 获取充值趋势
* `GET /api/report/consume` 获取消费趋势
* `GET /api/report/member-rank` 获取余额Top5
* `GET /api/report/worker-rank` 获取打手接单排行

---

### 🔹 企业微信推送 Service

* `POST /wecom/push` 内部调用企业微信 Webhook
* 三种消息模板：

  * 充值成功
  * 订单完成
  * 余额不足提醒
