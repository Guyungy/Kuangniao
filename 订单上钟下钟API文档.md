# 订单上钟下钟功能API文档

## 概述

订单上钟下钟功能用于管理订单的服务时间，包括：
- 上钟：记录订单开始服务时间
- 下钟：记录订单结束服务时间
- 自动计算下钟时间（基于上钟时间+服务时长）
- 获取即将下钟的订单列表（用于提醒）

## 数据库变更

### 新增字段

在 `orders` 表中新增以下字段：

```sql
-- 添加上钟时间字段
ALTER TABLE orders ADD COLUMN start_time DATETIME NULL COMMENT '上钟时间（开始服务时间）';

-- 添加下钟时间字段
ALTER TABLE orders ADD COLUMN end_time DATETIME NULL COMMENT '下钟时间（结束服务时间）';

-- 添加订单状态字段
ALTER TABLE orders ADD COLUMN status ENUM('pending', 'in_service', 'completed', 'cancelled') NOT NULL DEFAULT 'pending' COMMENT '订单状态';
```

### 字段说明

- `start_time`: 上钟时间，记录订单开始服务的时间
- `end_time`: 下钟时间，记录订单结束服务的时间，可通过上钟时间+服务时长自动计算
- `status`: 订单状态，包括：
  - `pending`: 待上钟
  - `in_service`: 服务中
  - `completed`: 已完成
  - `cancelled`: 已取消

## API接口

### 1. 上钟接口

**接口地址：** `POST /api/v1/orders/:id/start`

**功能描述：** 设置订单开始服务时间

**请求参数：**
- `id` (path): 订单ID
- `start_time` (body, optional): 上钟时间，ISO8601格式，不传则使用当前时间

**请求示例：**
```json
{
  "start_time": "2025-08-23T10:30:00.000Z"
}
```

**响应示例：**
```json
{
  "code": "00000",
  "message": "上钟成功",
  "data": {
    "order_id": 1,
    "start_time": "2025-08-23T10:30:00.000Z",
    "end_time": "2025-08-23T12:30:00.000Z",
    "worker_status": "忙碌"
  }
}
```

**业务逻辑：**
1. 检查订单是否存在
2. 检查订单是否已上钟（不能重复上钟）
3. 检查打手状态是否为可用
4. 设置上钟时间（默认当前时间）
5. 自动计算下钟时间（上钟时间 + 服务时长）
6. 更新打手状态为忙碌
7. 保存订单

### 2. 下钟接口

**接口地址：** `POST /api/v1/orders/:id/end`

**功能描述：** 设置订单结束服务时间

**请求参数：**
- `id` (path): 订单ID
- `end_time` (body, optional): 下钟时间，ISO8601格式，不传则使用当前时间

**请求示例：**
```json
{
  "end_time": "2025-08-23T12:30:00.000Z"
}
```

**响应示例：**
```json
{
  "code": "00000",
  "message": "下钟成功",
  "data": {
    "order_id": 1,
    "start_time": "2025-08-23T10:30:00.000Z",
    "end_time": "2025-08-23T12:30:00.000Z",
    "actual_duration": 2.0,
    "worker_status": "可用"
  }
}
```

**业务逻辑：**
1. 检查订单是否存在
2. 检查订单是否已上钟（未上钟无法下钟）
3. 检查订单是否已下钟（不能重复下钟）
4. 设置下钟时间（默认当前时间）
5. 更新打手状态为可用
6. 计算实际服务时长
7. 保存订单

### 3. 获取即将下钟订单列表

**接口地址：** `GET /api/v1/orders/ending-soon`

**功能描述：** 获取即将下钟的订单列表，用于提醒功能

**请求参数：**
- `minutes` (query, optional): 提醒时间范围（分钟），默认10分钟

**请求示例：**
```
GET /api/v1/orders/ending-soon?minutes=15
```

**响应示例：**
```json
{
  "code": "00000",
  "message": "获取即将下钟订单成功",
  "data": {
    "orders": [
      {
        "order_id": 1,
        "order_number": "ORD20250823000001",
        "member_name": "张三",
        "member_phone": "13800138000",
        "worker_name": "李四",
        "worker_phone": "13900139000",
        "start_time": "2025-08-23T10:30:00.000Z",
        "end_time": "2025-08-23T12:30:00.000Z",
        "remaining_minutes": 5,
        "duration": 2.0
      }
    ],
    "count": 1,
    "threshold_minutes": 15
  }
}
```

**业务逻辑：**
1. 查询已上钟但未下钟的订单
2. 筛选出在指定时间范围内即将下钟的订单
3. 计算每个订单的剩余时间
4. 返回订单列表和统计信息

## 前端功能

### 订单列表页面

1. **新增列：**
   - 状态列：显示订单当前状态（待上钟/服务中/已完成/已取消）
   - 上钟时间列：显示订单开始服务时间
   - 下钟时间列：显示订单结束服务时间

2. **新增按钮：**
   - 上钟按钮：仅对"待上钟"状态订单显示，点击后记录当前时间为上钟时间
   - 下钟按钮：仅对"服务中"状态订单显示，点击后记录当前时间为下钟时间

3. **状态显示：**
   - 待上钟：灰色标签，显示"待上钟"
   - 服务中：橙色标签，显示"服务中"，行背景高亮
   - 已完成：绿色标签，显示"已完成"
   - 已取消：红色标签，显示"已取消"

4. **视觉优化：**
   - 服务中订单行背景为橙色高亮
   - 左侧橙色边框标识服务中状态
   - 鼠标悬停时加深高亮效果

### 时间格式

所有时间显示采用北京时间（UTC+8），格式：`YYYY-MM-DD HH:mm:ss`

## 提醒功能预留

### 下钟提醒API

**接口地址：** `GET /api/v1/orders/ending-soon`

**使用场景：**
- 定时任务调用此接口获取即将下钟的订单
- 发送短信、微信或其他通知给相关人员
- 在管理界面显示提醒信息

**集成建议：**
1. 设置定时任务，每5分钟调用一次
2. 对即将下钟的订单发送提醒通知
3. 可在管理界面添加提醒面板
4. 支持自定义提醒时间范围

## 错误处理

### 常见错误码

- `B0001`: 参数验证失败
- `B0001`: 订单不存在
- `B0001`: 订单已上钟，不能重复上钟
- `B0001`: 订单未上钟，无法下钟
- `B0001`: 订单已下钟，不能重复下钟
- `B0001`: 打手当前不可用，无法上钟

### 错误响应格式

```json
{
  "code": "B0001",
  "message": "错误描述",
  "data": null
}
```

## 注意事项

1. **时间处理：** 所有时间都使用UTC时间存储，前端显示时转换为北京时间
2. **状态同步：** 上钟时自动将打手状态设为忙碌，下钟时自动设为可用
3. **数据一致性：** 确保订单状态、打手状态、时间字段的一致性
4. **权限控制：** 只有管理员可以执行上钟下钟操作
5. **日志记录：** 建议记录所有上钟下钟操作的日志，便于审计
