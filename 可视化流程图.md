# 🏊 泳道图（泳道：管理员 / 系统 / 企业微信）

```mermaid
flowchart LR
    A[管理员: 新增充值] --> B[系统: 校验会员存在]
    B --> C[系统: 更新余额 & 写入充值表]
    C --> D[系统: 更新会员累计充值]
    D --> E[系统: 调用企业微信Webhook]
    E --> F[企业微信群: 播报充值成功消息]

    G[管理员: 新增订单] --> H[系统: 校验会员 & 打手存在]
    H --> I[系统: 计算原价 (时长*单价)]
    I --> J{管理员修改金额?}
    J -->|是| K[系统: 记录优惠金额 & 原因]
    J -->|否| L[系统: 保持原价]

    K --> M{支付方式?}
    L --> M{支付方式?}

    M -->|余额| N[系统: 扣减余额]
    M -->|扫码| O[系统: 写入扫码支付流水]
    M -->|混合| P[系统: 扣余额 + 记录扫码补差]

    N --> Q[系统: 更新会员累计消费]
    O --> Q
    P --> Q

    Q --> R[系统: 写入订单表]
    R --> S[系统: 调用企业微信Webhook]
    S --> T[企业微信群: 播报订单完成消息]

    U[系统定时任务] --> V{会员余额<阈值?}
    V -->|是| W[系统: 调用企业微信Webhook]
    W --> X[企业微信群: 播报余额不足提醒]
```

---

# 📌 流程说明

### 1. 充值流程

* 管理员录入 → 系统更新余额、充值流水 → 调用企业微信 → 群播报「充值成功」。

### 2. 下单流程

* 管理员录入订单 → 系统自动算原价
* 管理员可改价 → 系统记录优惠 & 原因
* 支付方式分支：

  * 余额支付 → 扣余额
  * 扫码支付 → 只记账
  * 混合支付 → 扣余额 + 扫码补差额
* 系统写入订单表 & 更新消费金额 → 群播报「订单完成」。

### 3. 自动提醒

* 定时任务（每天扫一遍）
* 检测会员余额 < 阈值 → 群播报「余额不足提醒」。

---

# 🎯 用途

* 丢给开发：明确接口调用点（新增充值、新增订单、定时任务）。
* 丢给设计：可以直接转成 Figma / ProcessOn 的泳道图。
* 丢给运营：一看就知道群里会播报什么消息、什么时机播报。
